# AWS Free Tier Deployment Guide - Portfolio Stack

This guide details the step-by-step process of deploying your portfolio frontend and Spring Boot backend to the AWS Free Tier.

Since the Contact Us microservice dispatches emails via SMTP and does not store records in a database, we do not need AWS RDS, keeping your entire setup **100% free** under the AWS Free Tier.

---

## 🏗️ Architecture Design

* **React Frontend**: Deployed on **AWS Amplify** (directly connected to your GitHub repo for automatic CI/CD).
* **Spring Boot Backend**: Deployed on **AWS Elastic Beanstalk** (running a secure `t2.micro` or `t3.micro` Java instance).

---

## 📤 Step 1: Package the Backend for Deployment

First, package your Spring Boot backend into an executable `.jar` file:

1. Open your terminal in the `backend/` directory.
2. Run the clean package command:
   ```bash
   mvn clean package -DskipTests
   ```
3. This will create a file named `backend-0.0.1-SNAPSHOT.jar` inside the `backend/target/` directory. This is the bundle we will upload to AWS.

---

## ☁️ Step 2: Deploy Backend to AWS Elastic Beanstalk (Free)

Elastic Beanstalk automatically provisions and configures a free `t2.micro` server to run your Java app.

1. Open the [AWS Console](https://aws.amazon.com/) and navigate to **Elastic Beanstalk**.
2. Click **Create Application**.
3. Fill out the application configurations:
   * **Application Name**: `Portfolio-Backend`
   * **Platform**: Select **Java**.
   * **Platform Branch**: Select **Corretto 17** (or the version of Java you are running).
   * **Application Code**: Select **Upload your code** and upload the `backend-0.0.1-SNAPSHOT.jar` file from your `backend/target/` directory.
4. Click **Configure more options** (or **Next** depending on the AWS UI version).
5. Navigate to the **Environment Properties / Configuration** section. Under **Software / Environment properties**, add your environment variables:
   * `GMAIL_USERNAME` = `rajajaiswal8414@gmail.com`
   * `GMAIL_PASSWORD` = `your-16-char-app-password`
   * `ADMIN_EMAIL` = `rajajaiswal8414@gmail.com`
   * `CORS_ALLOWED_ORIGINS` = `https://main.dXXXXXXXXX.amplifyapp.com` *(You will update this with your React Amplify URL after Step 3)*
6. Click **Submit/Create**. AWS will take 2–3 minutes to set up your server and provide you with a public URL (e.g. `http://portfolio-backend.eba-xxxxxx.us-east-1.elasticbeanstalk.com`).

---

## 🎨 Step 3: Deploy Frontend to AWS Amplify (Free)

AWS Amplify is the easiest, most robust way to host Vite/React sites on AWS for free.

1. Open the AWS Console and navigate to **AWS Amplify**.
2. Click **New App** -> **Host web app**.
3. Select **GitHub** and authorize Amplify to access your GitHub repositories.
4. Select your **Portfolio** repository and the `main` branch.
5. In the Build settings, click **Advanced Settings** and add the environment variable:
   * Key: `VITE_API_URL`
   * Value: `http://portfolio-backend.eba-xxxxxx.us-east-1.elasticbeanstalk.com` *(Paste the URL generated by Elastic Beanstalk in Step 2)*
6. In **Build Settings / Monorepo Config**, make sure the root directory of the build points to `portfolio`:
   * Set **App root** to `portfolio`.
7. Click **Save and Deploy**. AWS Amplify will pull your repository, build the React production bundle, and deploy it to a secure `https://...amplifyapp.com` link.

---

## 🔄 Step 4: Link CORS on the Backend

Now that your frontend Amplify link is live, you must tell the backend server to accept requests from it:

1. Copy your Amplify URL (e.g. `https://main.d12345abcdef.amplifyapp.com`).
2. Go back to your **Elastic Beanstalk** environment -> **Configuration** -> **Software**.
3. Update or add the property:
   * Key: `CORS_ALLOWED_ORIGINS`
   * Value: `https://main.d12345abcdef.amplifyapp.com`
4. Apply changes. Your backend will restart automatically, and your full-stack contact form will now securely process inquiries from your live AWS URL!
